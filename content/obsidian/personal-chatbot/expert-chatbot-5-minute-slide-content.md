## Slide 2: Domain and Problem

### Slide Title

Selected Domain and Problem

### Slide Content

- Selected domain: Personal portfolio website assistant
- Target users:
  - Recruiters
  - Employers
  - Lecturers
  - Project reviewers
- Problem:
  - Portfolio information can be spread across many pages
  - Visitors may not quickly find the details they need
  - A chatbot can guide users to relevant information faster

### Speaking Script

The selected domain is a personal portfolio website assistant. The target users include recruiters, employers, lecturers, and people reviewing projects. A common problem with portfolio websites is that information can be spread across different pages, such as about, projects, skills, and blog posts. The chatbot helps users find relevant information quickly through conversation.

## Slide 3: Approach and Data Source

### Slide Title

Chatbot Approach and Knowledge Source

### Slide Content

- Approach: Retrieval-based or RAG-style chatbot
- Knowledge source:
  - Portfolio website content
  - Markdown or Obsidian notes
  - Project descriptions
  - Prepared question-answer examples
- Answer flow:
  - User asks a question
  - Chatbot searches relevant portfolio information
  - Chatbot generates a clear answer
  - If information is missing, chatbot gives a fallback response

### Speaking Script

For the approach, we use a retrieval-based or RAG-style chatbot. Instead of making up answers, the chatbot should use portfolio-related content such as website pages, Markdown notes, project descriptions, and prepared question-answer examples. When the user asks a question, the chatbot searches for relevant information, creates a clear answer, and gives a fallback response if the information is not available.

## Slide 6: Testing, Limitations, and Future Work

### Slide Title

Testing, Limitations, and Future Work

### Slide Content

- Testing included:
  - Greeting questions
  - Domain-specific questions
  - Project questions
  - Challenging questions
  - Out-of-scope questions
- Limitations:
  - Depends on the quality of the portfolio content
  - May not answer missing information
  - Needs more testing with real users
- Future work:
  - Add more knowledge sources
  - Improve answer accuracy
  - Add source citations
  - Improve the user interface

### Speaking Script

We tested the chatbot using different question types, including greetings, domain-specific questions, project questions, challenging questions, and out-of-scope questions. The main limitation is that the chatbot depends on the quality and completeness of the portfolio content. If the information is missing, the chatbot should not guess. In the future, we can improve it by adding more knowledge sources, adding citations, improving accuracy, and making the interface better.

## Short Closing Statement

This expert chatbot shows how a personal portfolio can become more interactive and easier to explore. It helps users ask direct questions and receive relevant answers about skills, projects, and background. The prototype works for the selected domain, but it can be improved with more complete data, better testing, and a stronger user interface.
