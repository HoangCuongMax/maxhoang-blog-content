# Expert Chatbot Presentation Task Division

## Purpose

This note divides the expert chatbot presentation preparation across 3 team members.

The selected domain is:

> Personal portfolio website assistant

The presentation should explain the chatbot idea, the selected domain, the technical approach, the data source, testing results, live demonstration, limitations, and future improvements.

## Team Role Summary

| Member | Main responsibility | Main output |
| --- | --- | --- |
| Member 1: Max | Build the chatbot and prepare the live demo | Working chatbot prototype, demo questions, technical explanation |
| Member 2 | Prepare introduction, domain, and approach slides | Slides for project background, domain, data source, model approach |
| Member 3 | Prepare testing, evaluation, conclusion, and future work | Test questions, response table, evaluation summary, final slides |

## Member 1: Max - Chatbot Builder and Demo Lead

### Main Goal

Build a working expert chatbot prototype for the personal portfolio website assistant domain and prepare the live demonstration.

### Tasks

- [ ] Confirm the chatbot domain: personal portfolio website assistant.
- [ ] Decide whether the chatbot uses website content, Obsidian notes, prepared Q&A data, or a RAG approach.
- [ ] Prepare the knowledge source from the portfolio website or Markdown/Obsidian content.
- [ ] Clean and structure the chatbot knowledge data.
- [ ] Build the chatbot interface or demo environment.
- [ ] Implement greeting and simple conversation handling.
- [ ] Implement domain-specific question answering.
- [ ] Implement fallback handling for unknown or out-of-scope questions.
- [ ] Reduce hallucination by making the chatbot answer only from supported information.
- [ ] Prepare 3 to 5 live demo questions.
- [ ] Test the live demo before the presentation.

### Demo Questions to Prepare

- [ ] Simple greeting question.
- [ ] Question about Max's skills or background.
- [ ] Question about Max's projects.
- [ ] Challenging question that requires a useful but careful answer.
- [ ] Out-of-scope question to show fallback handling.

### Slides or Speaking Points

- How the chatbot was built.
- What model or method was used.
- What data or knowledge source was used.
- How a user question becomes a chatbot answer.
- What the chatbot can do.
- What the chatbot cannot do.
- Live chatbot demonstration.

## Member 2 - Introduction, Domain, and Approach Lead

### Main Goal

Prepare the beginning and middle part of the presentation so the audience understands the problem, selected domain, and technical approach.

### Tasks

- [ ] Explain what an expert chatbot is.
- [ ] Introduce the selected domain: personal portfolio website assistant.
- [ ] Explain why this domain is useful.
- [ ] Define the target users.
- [ ] Explain the main problems the chatbot helps solve.
- [ ] Explain whether the project connects to ABSA or uses a different approach.
- [ ] Explain the selected chatbot approach.
- [ ] Explain why this approach was selected.
- [ ] Explain the data or knowledge source.
- [ ] Explain data preparation or cleaning.
- [ ] Prepare slides for introduction, domain, data source, and approach.

### Slides or Speaking Points

- Expert chatbot definition.
- Selected domain.
- Target users.
- Domain challenge.
- Why a chatbot is useful for a personal portfolio website.
- Knowledge source.
- Model or chatbot approach.
- Development workflow overview.

## Member 3 - Testing, Evaluation, Conclusion, and Future Work Lead

### Main Goal

Prepare the evidence that the chatbot works, explain its limitations, and close the presentation clearly.

### Tasks

- [ ] Prepare test questions for the chatbot.
- [ ] Include simple conversation tests.
- [ ] Include domain-specific questions.
- [ ] Include difficult or complex questions.
- [ ] Include out-of-scope questions.
- [ ] Record chatbot responses.
- [ ] Evaluate whether the answers are correct and useful.
- [ ] Identify common chatbot mistakes.
- [ ] Prepare a testing results table.
- [ ] Explain what worked well.
- [ ] Explain the main limitations.
- [ ] Suggest future improvements.
- [ ] Prepare slides for testing, evaluation, conclusion, and future work.
